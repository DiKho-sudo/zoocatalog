# 📘 Полная инструкция по работе с 1С Розница

## 🎯 У вас есть 2 файла из 1С:

### 1. **про.xlsx** - Вся номенклатура
- Многоуровневая структура категорий
- Артикулы, наименования, характеристики
- БЕЗ цен и остатков

- Остаток на складе
- Стоимость (в розничных ценах)
- Себестоимость

## 🔄 Регулярное обновление каталога

### Каждый день / неделю:

```bash
# 1. Обновите остатки и цены из нового отчета 1С
python manage.py sync_stock_prices "остаток_цена.xlsx"

# 2. Если добавились новые товары - импортируйте номенклатуру
python manage.py import_from_1c "про.xlsx" --update

# 3. Проверьте результат на сайте
# http://localhost:3000/catalog
```

## 📋 Что делают команды:

### `import_from_1c` (первичный импорт)
✅ Создает товары из про.xlsx  
✅ Создает категории и бренды  
✅ Обрабатывает многоуровневую структуру  
⚠️ Цены будут 0 (обновите через sync_stock_prices)  
⚠️ Статус "В наличии" (обновите через sync_stock_prices)  

### `sync_stock_prices` (обновление данных)
✅ Обновляет цены из колонки "Стоимость"  
✅ Обновляет статус наличия по остаткам  
✅ Ищет товары по названию  
✅ Показывает что изменилось  

### `add_images` (добавление картинок)
✅ Добавляет изображения по названию файла  
✅ Поддерживает массовую загрузку  
✅ Можно запускать многократно

## 🎨 Работа с изображениями

Вы сказали что будете добавлять сами - вот лучшие способы:

### Способ 1: По одному через админку
1. http://localhost:8000/admin/products/product/
2. Найдите товар (есть поиск!)
3. Нажмите на название
4. В поле "Image" выберите файл
5. Сохраните

**Плюсы:** Видно результат сразу  
**Минусы:** Долго для 250 товаров

### Способ 2: Массовая загрузка
1. Соберите картинки в папку `images_zoo`
2. Назовите файлы ключевыми словами из товара:
   ```
   ecoline.jpg → найдет "ECOLINE ROMEO GREEN..."
   совок.jpg → найдет "Совок для корма..."
   lux.jpg → найдет "Туалет LUX..."
   ```
3. Запустите:
   ```bash
   python manage.py add_images ../images_zoo --by-name
   ```

**Плюсы:** Быстро для большого количества  
**Минусы:** Нужно правильно называть файлы

## ⚙️ Как это работает технически:

### Многоуровневые категории
Скрипт автоматически создает иерархию:
```
Корма
├── Корма для собак
│   ├── Сухой корм
│   └── Влажный корм
└── Корма для кошек
    ├── Сухой корм
    └── Влажный корм
```

Из вашего файла берется "Вид номенклатуры" или подобная колонка.

### Сопоставление товаров
При синхронизации цен ищет товар:
1. Точное совпадение названия
2. Совпадение первых слов
3. Частичное совпадение (первые 20 символов)

### Определение статуса наличия
```
Остаток > 0 → "В наличии" (in_stock)
Остаток = 0 → "Нет в наличии" (out_of_stock)
```

## ❓ Частые вопросы:

**Q: Нужно ли каждый раз импортировать про.xlsx?**  
A: Нет! Только первый раз или если добавились новые товары. Для обновления цен используйте только sync_stock_prices.

**Q: Что если названия не совпадают?**  
A: Команда ищет по частичному совпадению. Если не находит - покажет список.

**Q: Как часто обновлять?**  
A: Цены и остатки - хоть каждый день. Номенклатуру - только если появились новые товары.

**Q: Можно ли отменить изменения?**  
A: Используйте флаг --dry-run для предпросмотра перед реальным обновлением.
