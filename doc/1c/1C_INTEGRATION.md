# Интеграция с 1С Розница 8.3 для Беларуси

## 📋 Шаг 1: Выгрузка товаров из 1С Розница

### Вариант A: Стандартная выгрузка (рекомендуется для начала)

1. **Откройте 1С: Розница 8.3**

2. **Перейдите в справочник номенклатуры:**
   - Главное меню → **"Товары"** → **"Номенклатура"**

3. **Настройте список для выгрузки:**
   - Нажмите кнопку **"Еще"** (или правая кнопка мыши)
   - Выберите **"Изменить форму списка"**
   - Добавьте нужные колонки:
     - ✅ Наименование (обязательно)
     - ✅ Артикул (обязательно для обновлений)
     - ✅ Цена продажи
     - ✅ Группа номенклатуры
     - ✅ Производитель (если есть реквизит)
     - ✅ Остаток
     - ✅ Вес
     - ✅ Комментарий/Описание

4. **Выгрузите список:**
   - Нажмите **"Все действия"** → **"Вывести список"**
   - В табличном документе: **Файл** → **"Сохранить как"**
   - Выберите формат: **Microsoft Excel (.xlsx)**
   - Сохраните файл, например: `товары_из_1с.xlsx`

### Вариант B: Через отчет (если настроен)

1. Перейдите в **"Отчеты"** → **"Остатки товаров"**
2. Настройте параметры отчета
3. Сформируйте отчет
4. Сохраните в Excel

### Пример структуры файла Excel:

| Наименование | Артикул | Цена | Группа | Производитель | Остаток | Вес |
|-------------|---------|------|---------|---------------|---------|-----|
| Royal Canin для собак | RC001 | 45.90 | Корма для собак | Royal Canin | 15 | 1.5 |
| Whiskas влажный корм | WH002 | 3.20 | Корма для кошек | Whiskas | 50 | 0.085 |

## 📸 Шаг 2: Подготовка изображений товаров

### Где взять изображения из 1С:

1. **Если изображения хранятся в 1С:**
   - Откройте карточку товара
   - Правой кнопкой на изображении → **"Сохранить картинку как"**
   - Сохраните с именем = **Артикул товара**

2. **Автоматическая выгрузка изображений:**
   - В 1С есть возможность массовой выгрузки
   - Используйте обработку выгрузки файлов (если есть)

3. **Структура папки с изображениями:**
```
images_1c/
├── RC001.jpg         (изображение для артикула RC001)
├── WH002.png         (изображение для артикула WH002)
├── FX003.jpg
└── ...
```

**Важно:**
- Имя файла = Артикул товара из 1С
- Поддерживаемые форматы: .jpg, .jpeg, .png, .webp
- Рекомендуемый размер: 800x800px или больше

## 🚀 Шаг 3: Импорт в Django

### Установка зависимостей:

```bash
cd backend
pip install openpyxl python-dotenv psycopg2-binary
```

### Базовый импорт (без изображений):

```bash
python manage.py import_from_1c путь/к/товары_из_1с.xlsx
```

### Импорт с изображениями:

```bash
python manage.py import_from_1c товары_из_1с.xlsx --images-dir путь/к/images_1c
```

### Обновление существующих товаров:

```bash
python manage.py import_from_1c товары_из_1с.xlsx --update --images-dir images_1c
```

### Примеры использования:

```bash
# Windows - импорт с изображениями
python manage.py import_from_1c D:\1c_data\товары.xlsx --images-dir D:\1c_data\images

# Обновить цены и остатки
python manage.py import_from_1c D:\1c_data\товары.xlsx --update

# Только создать новые товары (не обновлять)
python manage.py import_from_1c товары.xlsx
```

## 📊 Что делает команда импорта:

✅ **Автоматически создает:**
- Категории (из колонки "Группа")
- Производителей (из колонки "Производитель")
- Товары со всеми характеристиками

✅ **Определяет статус наличия:**
- Если остаток > 0 → "В наличии"
- Если остаток = 0 → "Нет в наличии"

✅ **Обрабатывает изображения:**
- Ищет файл по артикулу
- Автоматически загружает в медиа-хранилище Django
- Поддерживает разные форматы

✅ **Показывает статистику:**
- Сколько товаров создано
- Сколько обновлено
- Сколько пропущено
- Ошибки импорта

## 🔄 Регулярная синхронизация

### Вручную:

1. Выгрузите обновленный файл из 1С
2. Запустите команду импорта с флагом `--update`
3. Проверьте результаты

### Автоматизация (опционально):

Создайте bat-файл `sync_1c.bat`:

```batch
@echo off
echo Синхронизация с 1С...
cd D:\courseProj\backend
python manage.py import_from_1c D:\1c_data\товары.xlsx --update --images-dir D:\1c_data\images
echo Готово!
pause
```

Запускайте его когда нужно обновить каталог.

## ⚙️ Настройка PostgreSQL (опционально, но рекомендуется)

### Зачем PostgreSQL:
- ✅ Быстрее для больших каталогов
- ✅ Лучший полнотекстовый поиск
- ✅ Надежнее для продакшн

### Установка:

1. **Скачайте PostgreSQL:**
   https://www.postgresql.org/download/windows/

2. **Установите** (запомните пароль для postgres)

3. **Создайте базу данных:**
   - Откройте pgAdmin
   - Правой кнопкой на "Databases" → Create → Database
   - Имя: `zoo_catalog`

4. **Настройте Django:**
   ```bash
   cd backend
   # Создайте .env файл
   echo DB_ENGINE=django.db.backends.postgresql > .env
   echo DB_NAME=zoo_catalog >> .env
   echo DB_USER=postgres >> .env
   echo DB_PASSWORD=ваш_пароль >> .env
   echo DB_HOST=localhost >> .env
   echo DB_PORT=5432 >> .env
   ```

5. **Примените миграции:**
   ```bash
   python manage.py migrate
   python manage.py createsuperuser
   ```

Подробнее: см. `POSTGRESQL_SETUP.md`

## ❓ Частые вопросы

**Q: Что делать если колонки в Excel называются по-другому?**  
A: Команда автоматически распознает разные варианты названий. Если не работает, переименуйте колонки в Excel на русские стандартные.

**Q: Можно ли импортировать несколько раз?**  
A: Да! Используйте флаг `--update` для обновления существующих товаров.

**Q: Изображения не импортируются**  
A: Проверьте что имена файлов = артикулам товаров (без пробелов и спецсимволов).

**Q: Как часто обновлять каталог?**  
A: Рекомендуется 1 раз в день или после значительных изменений в 1С.

**Q: Что делать с товарами которых нет в новой выгрузке?**  
A: Они останутся в базе. Можно вручную удалить через админку или добавить логику удаления.
