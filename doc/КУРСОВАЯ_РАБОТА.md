## ВВЕДЕНИЕ

Данный курсовой проект представляет собой практическую реализацию веб-каталога зоомагазина — информационной системы для просмотра товаров с функциями поиска, фильтрации и рекомендаций. Система интегрирована с учётной программой 1С: Предприятие 8.3 и работает с реальными данными действующего зоомагазина.

Актуальность работы обусловлена потребностью малого бизнеса в доступных информационных ресурсах для представления ассортимента клиентам, без необходимости разработки полноценного интернет-магазина с платёжными системами и личными кабинетами.

Цель работы — разработка функционального веб-каталога с автоматической синхронизацией данных из 1С и системой рекомендаций товаров.

Задачи:
- реализовать серверную часть на Django с REST API;
- разработать клиентскую часть на React с адаптивным интерфейсом;
- реализовать интеграцию с 1С через импорт отчётов Excel;
- разработать алгоритм расчёта розничных цен для штучных и весовых товаров;
- реализовать систему рекомендаций на основе контентной фильтрации и статистики просмотров.

Технологический стек: Python/Django, Django REST Framework, React 18, PostgreSQL, Docker, openpyxl.

## ГЛАВА 1. АРХИТЕКТУРА И ТЕХНОЛОГИЧЕСКИЙ СТЕК

### 1.1 Общая архитектура системы

Система построена по архитектуре клиент-сервер с разделением на три уровня:

1. **Frontend** (React SPA) — выполняется в браузере, отправляет HTTP-запросы к API, управляет состоянием интерфейса.
2. **Backend** (Django + DRF) — предоставляет REST API, обрабатывает запросы фильтрации/поиска/рекомендаций, управляет импортом данных из 1С.
3. **База данных** (PostgreSQL) — хранит товары, категории, производителей, статистику просмотров.

Взаимодействие: пользователь открывает страницу → React отправляет `GET /api/products/` → Django применяет фильтры через ORM → PostgreSQL выполняет запрос → JSON-ответ → React обновляет DOM через виртуальный DOM.

Backend организован как Django-проект `zoo_catalog` с приложением `products`. Frontend организован по компонентному принципу: `components/`, `pages/`, `services/`.

### 1.2 Структура базы данных

Схема включает шесть таблиц:

| Таблица | Назначение | Ключевые поля |
|---------|-----------|---------------|
| `Category` | Иерархия категорий | `name`, `slug`, `parent_id` (самосвязь) |
| `Brand` | Производители | `name`, `country` |
| `AnimalType` | Типы животных | `name`, `slug` |
| `Product` | Товары | `name`, `price`, `weight`, `unit`, `view_count`, `stock_status` |
| `ProductImage` | Галерея изображений | `product_id`, `image`, `order` |

Связи: `Product` → `Category` (многие-к-одному), `Product` → `Brand` (многие-к-одному), `Product` → `AnimalType` (многие-к-одному), `ProductImage` → `Product` (многие-к-одному).

Поле `unit` (`шт`/`кг`/`г`/`л`/`мл`) определяет тип товара и алгоритм расчёта цены. Поле `view_count` накапливает статистику просмотров для системы рекомендаций.

### 2.1 Модели данных Django

Центральная модель `Product` содержит все атрибуты товара. Ключевые поля:

```python
class Product(models.Model):
    name = models.CharField(max_length=300)
    slug = models.SlugField(max_length=300, unique=True)
    description = models.TextField()
    composition = models.TextField(blank=True)
    weight = models.DecimalField(max_digits=10, decimal_places=3)
    unit = models.CharField(max_length=20, default='шт')
    price = models.DecimalField(max_digits=10, decimal_places=2)

    category = models.ForeignKey(Category, on_delete=models.CASCADE, related_name='products')
    brand = models.ForeignKey(Brand, on_delete=models.CASCADE, related_name='products')
    animal_type = models.ForeignKey(AnimalType, on_delete=models.CASCADE, related_name='products')

    view_count = models.PositiveIntegerField(default=0)
    stock_status = models.CharField(max_length=20, choices=STOCK_CHOICES, default='in_stock')

    class Meta:

        indexes = [
            models.Index(fields=['category', 'animal_type']),
            models.Index(fields=['price']),
            models.Index(fields=['-created_at']),
```

Индексы на `category + animal_type`, `price` и `created_at` ускоряют основные сценарии: фильтрацию по категории, сортировку по цене и отображение новых товаров.

### 2.2 REST API и сериализация

API построен на `ReadOnlyModelViewSet` с двумя сериализаторами:

Оптимизация запросов через `select_related` и `prefetch_related` сокращает количество SQL-запросов с 37 до 4 при загрузке списка из 12 товаров:

```python
class ProductViewSet(viewsets.ReadOnlyModelViewSet):
```

### 2.3 Система поиска и фильтрации

Фильтрация реализована через `django-filter`:

```python
class ProductFilter(django_filters.FilterSet):
```

Полнотекстовый поиск через `SearchFilter` ищет по полям `name`, `description`, `composition`. Django генерирует SQL с оператором `ILIKE` для регистронезависимого поиска.

## ГЛАВА 3. ИНТЕГРАЦИЯ С 1С: ПРЕДПРИЯТИЕ

### 3.1 Формат отчёта «Оценка склада»

Источником данных для каталога служит отчёт «Оценка склада (по категориям в розничных ценах)» из 1С: Предприятие 8.3. Отчёт имеет иерархическую структуру в одной колонке Excel-файла, где уровень вложенности определяется количеством пробелов в начале строки:

| Отступ | Уровень | Пример |
|--------|---------|--------|
| 0 пробелов | Склад или бренд | `Зоомагазин, Карпова, 12 ЧУП` или `ACANA` |
| 1+ пробелов | Товар | ` Корм BRIT Premium 15кг., , кг` |

Числовые колонки: **Остаток на складе** (F), **Себестоимость** (G), **Стоимость в розничных ценах** (H).

### 3.2 Парсер иерархического отчёта

Реализована Django management-команда `sync_stock_prices`, которая:

1. Автоматически определяет позиции колонок, сканируя заголовки в первых 20 строках.
2. Классифицирует каждую строку по типу на основе отступа и содержимого:

```python
def _detect_row_type(self, row, cfg):
    raw_str = str(row[cfg['text_col']])
    text = raw_str.strip()

    if 'итого' in text.lower():
        return 'total'

    indent = len(raw_str) - len(raw_str.lstrip())
    if indent > 0:
        return 'product'

    # indent == 0: склад, бренд или товар без отступа
    if any(k in text.lower() for k in ('магазин', 'чуп', 'ооо')):
        return 'warehouse'
    if text.startswith('<'):
        return 'brand'
    if re.search(r',\s*,?\s*(шт|кг|г|л|мл)\s*$', text):
        return 'product'
    if re.search(r'\d+[,.]?\d*\s*(кг|г|мл|л|таб)', text, re.IGNORECASE):
        return 'product'

    return 'brand'
```

3. При чтении запоминает текущий бренд и ассоциирует все последующие товары с ним.

### 3.3 Алгоритм расчёта розничных цен

Колонка «Стоимость (в розничных ценах)» содержит **суммарную** стоимость всего остатка, а не цену за единицу. Алгоритм расчёта зависит от единицы измерения:

**Штучные товары** (`шт`):

$$P_\text{шт} = \frac{C_\text{розн}}{Q}$$

**Весовые товары** (`кг`) — корма и лакомства:

$$P_\text{уп} = \frac{C_\text{розн}}{Q_\text{кг}} \times W_\text{уп}$$

где $W_\text{уп}$ — вес упаковки, извлекаемый из названия товара регулярным выражением:

```python
def _extract_package_weight(self, raw):
    kg = re.search(r'(\d+[,.]?\d*)\s*кг', raw)
    if kg:
        return float(kg.group(1).replace(',', '.'))

    g = re.search(r'(\d+[,.]?\d*)\s*г[.,\s]', raw)
    if g:
        return float(g.group(1).replace(',', '.')) / 1000.0

    return 1.0  # по умолчанию — цена за 1 кг
```

Пример расчёта: товар «Корм BRIT Premium, 15кг.» с остатком 42,757 кг и розничной стоимостью 690,95 BYN:

$$P_\text{кг} = 690{,}95 / 42{,}757 = 16{,}16 \text{ BYN/кг}$$
$$P_\text{уп} = 16{,}16 \times 15 = 242{,}39 \text{ BYN за мешок}$$

Для финансовых расчётов используется `Decimal` с банковским округлением `ROUND_HALF_UP` [4].

Результаты синхронизации на реальных данных (2056 строк):

| Показатель | Значение |
|-----------|---------|
| Обновлено товаров | 946 |
| Не найдено в базе | 200 |
| Пропущено (бренды, склад, итого) | 900 |

## ГЛАВА 4. СИСТЕМА РЕКОМЕНДАЦИЙ

### 4.1 Алгоритм контентной фильтрации

Для рекомендации похожих товаров реализован алгоритм взвешенной контентной фильтрации (content-based filtering) [18]. Для каждого товара-кандидата рассчитывается оценка релевантности (score) на основе совпадения атрибутов с текущим товаром:

| Критерий | Вес | Обоснование |
|----------|-----|-------------|
| Совпадение категории | +3 | Наиболее значимый фактор: товары одной категории взаимозаменяемы |
| Совпадение типа животного | +2 | Товары для одного животного релевантны владельцу |
| Совпадение бренда | +1 | Лояльность к бренду — распространённый паттерн |
| Цена в диапазоне ±50% | +1 | Товары аналогичного ценового сегмента |
| Оба гипоаллергенные | +2 | Владельцы аллергичных животных ищут специализированные товары |
| Оба беззерновые | +2 | Аналогично — диетические предпочтения устойчивы |

Максимальный score = 11 баллов. Критерии `is_hypoallergenic` и `is_grain_free` добавляются к score динамически — только если текущий товар сам обладает соответствующим признаком. Это предотвращает нерелевантное повышение score для обычных товаров:

```python
annotations = (
    Value(0, output_field=IntegerField())
    + Case(When(category=product.category, then=Value(3)),
           default=Value(0), output_field=IntegerField())
    + Case(When(animal_type=product.animal_type, then=Value(2)),
           default=Value(0), output_field=IntegerField())
    + Case(When(brand=product.brand, then=Value(1)),
           default=Value(0), output_field=IntegerField())
    + Case(When(price__gte=price_lo, price__lte=price_hi, then=Value(1)),
           default=Value(0), output_field=IntegerField())
)

if product.is_hypoallergenic:
    annotations = annotations + Case(
        When(is_hypoallergenic=True, then=Value(2)),
        default=Value(0), output_field=IntegerField())

if product.is_grain_free:
    annotations = annotations + Case(
        When(is_grain_free=True, then=Value(2)),
        default=Value(0), output_field=IntegerField())
```

Вычисление score выполняется **на уровне SQL** через `Case/When` аннотации Django ORM. PostgreSQL оптимизирует запрос с использованием индексов, что эффективнее загрузки всех товаров в Python [7].

### 4.2 Анонимные сессии и история просмотров

Для персонализации без регистрации реализована система анонимных сессий. При первом посещении сайта фронтенд генерирует уникальный идентификатор (UUID v4) и сохраняет его в `localStorage` браузера:

```javascript
function getSessionId() {
  let sid = localStorage.getItem('zoo_session_id');
  if (!sid) {
    sid = crypto.randomUUID();
    localStorage.setItem('zoo_session_id', sid);
  }
  return sid;
}
```

Идентификатор передаётся с каждым запросом к API через HTTP-заголовок `X-Session-ID`, настроенный через Axios interceptor:

```javascript
api.interceptors.request.use(config => {
  config.headers['X-Session-ID'] = getSessionId();
  return config;
});
```

На бэкенде создана модель `ProductView` для хранения истории просмотров:

```python
class ProductView(models.Model):
    session_id = models.CharField(max_length=64, db_index=True)
    product = models.ForeignKey(Product, on_delete=models.CASCADE, related_name='views')
    viewed_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        indexes = [
            models.Index(fields=['session_id', '-viewed_at']),
        ]
```

Составной индекс на `(session_id, viewed_at)` обеспечивает быструю выборку последних просмотров конкретного пользователя. При каждом открытии страницы товара эндпоинт `track-view` атомарно инкрементирует глобальный счётчик и записывает просмотр в историю сессии:

```python
@action(detail=True, methods=['post'], url_path='track-view')
def track_view(self, request, slug=None):
    product = self.get_object()
    Product.objects.filter(pk=product.pk).update(view_count=F('view_count') + 1)

    sid = request.headers.get('X-Session-ID', '')
    if sid:
        ProductView.objects.create(session_id=sid, product=product)

    return Response({'status': 'ok'})
```

Использование `F('view_count') + 1` обеспечивает атомарность — корректный инкремент при конкурентных запросах без race condition [1].

### 4.3 Персональные рекомендации

На основе истории просмотров реализован эндпоинт `GET /api/products/recommended/`, который строит профиль предпочтений пользователя и подбирает товары. Алгоритм:

1. **Извлечение истории** — последние 30 просмотров сессии.
2. **Построение профиля** — подсчёт частоты встречаемости категорий, типов животных и брендов среди просмотренных товаров; определение интереса к гипоаллергенным и беззерновым товарам.
3. **Скоринг кандидатов** — score рассчитывается аналогично `similar`, но на основе агрегированного профиля: топ-3 категории, топ-2 типа животных, топ-3 бренда.
4. **Исключение просмотренных** — уже просмотренные товары не рекомендуются повторно.

```python
@action(detail=False, methods=['get'], url_path='recommended')
def recommended(self, request):
    sid = request.headers.get('X-Session-ID', '')
    if not sid:
        return Response([])

    recent_views = ProductView.objects.filter(session_id=sid) \
        .select_related('product').order_by('-viewed_at')[:30]

    # Строим профиль предпочтений
    viewed_ids, cat_counts, animal_counts, brand_counts = set(), {}, {}, {}
    has_hypo, has_gf = False, False

    for pv in recent_views:
        p = pv.product
        viewed_ids.add(p.pk)
        cat_counts[p.category_id] = cat_counts.get(p.category_id, 0) + 1
        animal_counts[p.animal_type_id] = animal_counts.get(p.animal_type_id, 0) + 1
        brand_counts[p.brand_id] = brand_counts.get(p.brand_id, 0) + 1
        if p.is_hypoallergenic: has_hypo = True
        if p.is_grain_free: has_gf = True

    top_cats = sorted(cat_counts, key=cat_counts.get, reverse=True)[:3]
    top_animals = sorted(animal_counts, key=animal_counts.get, reverse=True)[:2]
    top_brands = sorted(brand_counts, key=brand_counts.get, reverse=True)[:3]

    # Скоринг кандидатов по профилю
    annotations = (
        Value(0, output_field=IntegerField())
        + Case(When(category_id__in=top_cats, then=Value(3)), ...)
        + Case(When(animal_type_id__in=top_animals, then=Value(2)), ...)
        + Case(When(brand_id__in=top_brands, then=Value(1)), ...)
    )
    if has_hypo:
        annotations += Case(When(is_hypoallergenic=True, then=Value(2)), ...)
    if has_gf:
        annotations += Case(When(is_grain_free=True, then=Value(2)), ...)

    qs = Product.objects.exclude(pk__in=viewed_ids) \
        .filter(stock_status='in_stock') \
        .annotate(score=annotations).filter(score__gt=0) \
        .order_by('-score', '-view_count')[:8]
```

Таким образом, пользователь, который просматривал беззерновые корма для собак бренда ACANA, увидит на главной странице рекомендации: другие беззерновые корма для собак ACANA (score 8), беззерновые корма других брендов для собак (score 7), обычные корма для собак (score 5) и т.д. — без регистрации и авторизации.

### 5.3 Интеграция рекомендаций в интерфейс

**Детальная страница товара** загружает три ресурса параллельно при монтировании:

```javascript
useEffect(() => {
    getProductBySlug(slug).then(r => { setProduct(r.data); setLoading(false); });
    trackProductView(slug).catch(() => {});
    getSimilarProducts(slug).then(r => setSimilarProducts(r.data || []));
}, [slug]);
```

Блок «Похожие товары» отображается только при наличии результатов:

```jsx
{similarProducts.length > 0 && (
  <section className="similar-products-section">
    <h2>Похожие товары</h2>
    <div className="similar-products-grid">
      {similarProducts.map(p => <ProductCard key={p.id} product={p} />)}
    </div>
  </section>
)}
```

**Главная страница** загружает популярные товары параллельно с остальными данными через `Promise.all`:

```javascript
Promise.all([
    getProducts({ page_size: 8 }),
    getCategories(),
    getPopularProducts()
]).then(([productsRes, categoriesRes, popularRes]) => {
    setLatestProducts(productsRes.data.results || []);
    setCategories((categoriesRes.data.results || []).slice(0, 6));
    setPopularProducts(popularRes.data || []);
});
```

Адаптивная сетка рекомендаций: 4 колонки на десктопе, 2 на планшете, 1 на мобильных.

| Эндпоинт | Метод | Результат |
|-----------|-------|----------|

**Система импорта — реальный отчёт из 1С (2056 строк):**

- Корректно определены 106 брендов из иерархии отчёта;
- Расчёт цен для штучных товаров: 148,32 / 18 = 8,24 BYN (верно);
- Расчёт цен для весовых товаров: (95,32 / 2,509) × 11,4 = 433,12 BYN за упаковку (верно);
- Режим `--dry-run` позволяет проверить изменения без сохранения.

**Система рекомендаций:**

- Для беззернового корма ACANA для собак рекомендуются: другие беззерновые корма ACANA для собак (score 9), беззерновые корма других брендов для собак (score 8), обычные корма для собак (score 5) — критерии `is_grain_free` и `is_hypoallergenic` корректно учитываются;
- Персональные рекомендации: после просмотра 3–5 товаров на главной появляется блок «Рекомендуем для вас» с товарами, подобранными на основе истории просмотров;

- Среднее время ответа эндпоинтов `similar` и `recommended`: < 50 мс благодаря вычислению score на уровне SQL.

## ЗАКЛЮЧЕНИЕ

В результате выполнения курсового проекта разработан полнофункциональный веб-каталог зоомагазина, включающий:

1. **REST API** на Django с оптимизированными запросами, полнотекстовым поиском и гибкой фильтрацией по 8 параметрам.

2. **Интеграцию с 1С: Предприятие** — автоматический импорт данных из иерархического отчёта «Оценка склада» с дифференцированным расчётом цен для штучных и весовых товаров. Система обработала 2056 строк реального отчёта, обновив 946 товаров.

3. **Систему рекомендаций** из двух уровней: контентная фильтрация «Похожие товары» (скоринг по категории +3, типу животного +2, бренду +1, цене +1, гипоаллергенности +2, беззерновости +2) и персональные рекомендации «Рекомендуем для вас» на основе анонимных сессий и истории просмотров. Вычисление score выполняется на уровне SQL.

4. **Адаптивный интерфейс** на React 18 с блоками «Похожие товары» и «Популярные товары», корректным отображением цен за упаковку для весовых товаров.

5. **Контейнеризацию** через Docker Compose для упрощения развёртывания.

Практическая значимость подтверждается работой с реальными данными из учётной системы действующего зоомагазина.

Направления развития:
- автоматическая синхронизация с 1С через веб-сервисы;
- система отзывов и рейтингов;
- коллаборативная фильтрация (матрица совместных просмотров);
- A/B-тестирование весов скоринга для оптимизации рекомендаций.
