## 4. REST API (базовый префикс `/api/`)

Все ресурсы — **только чтение** (`ReadOnlyModelViewSet`), кроме `POST` для учёта просмотра.

### 4.1. Роутер DRF

| Ресурс | Базовый URL | Примечание |
|--------|-------------|------------|
| Типы животных | `/api/animal-types/` | Деталь: по `slug` |
| Бренды | `/api/brands/` | Поиск `?search=`, сортировка `?ordering=` |
| Категории | `/api/categories/` | Деталь: по `slug` |
| Товары | `/api/products/` | Деталь: по `slug` |

### 4.2. Товары: фильтры и поиск

Параметры запроса к `/api/products/` (часть реализована в `ProductFilter`):

| Параметр | Назначение |
|----------|------------|
| `price_min`, `price_max` | Диапазон цены |
| `brand` | ID бренда |
| `category` | ID категории |
| `animal_type` | ID типа животного |
| `age_group` | Возрастная группа |
| `is_hypoallergenic`, `is_grain_free` | Булевы фильтры |
| `stock_status` | Наличие |
| `search` | Поиск по названию, описанию, составу |
| `ordering` | Сортировка: `price`, `created_at`, `name`, `view_count` |
| `page`, `page_size` | Пагинация (размер страницы по умолчанию 12) |

### 4.3. Действия рекомендаций (`ProductViewSet`)

| Метод | URL | Описание |
|-------|-----|----------|
| `POST` | `/api/products/{slug}/track-view/` | Увеличить `view_count`, записать `ProductView` при наличии `X-Session-ID` |
| `GET` | `/api/products/{slug}/similar/` | До 8 похожих товаров (контентный скоринг) |
| `GET` | `/api/products/popular/` | До 8 популярных по `view_count` |
| `GET` | `/api/products/recommended/` | До 8 персональных рекомендаций по истории сессии |

**Заголовок клиента:** `X-Session-ID` — UUID сессии из `localStorage`; для CORS заголовок должен быть разрешён на сервере (`CORS_ALLOW_HEADERS` в `settings.py`).

### 4.4. Алгоритм скоринга «похожие» (кратко)

- Совпадение категории: +3  
- Тип животного: +2  
- Бренд: +1  
- Цена в диапазоне 50–200% от цены текущего товара: +1  
- При `is_hypoallergenic` / `is_grain_free` у текущего товара — +2 за совпадение у кандидата  

Исключаются сам товар и позиции не в наличии; сортировка по `score`, затем по `view_count`.

## 6. Переменные окружения (backend)

| Переменная | Назначение |
|------------|------------|
| `SECRET_KEY` | Секретный ключ Django |
| `DEBUG` | `True`/`False` |
| `ALLOWED_HOSTS` | Список через запятую (в проде — домены) |
| `DATABASE_URL` | Строка подключения (например, от Render PostgreSQL) |
| Без `DATABASE_URL` | Используется SQLite **или** параметры `DB_ENGINE`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_HOST`, `DB_PORT` для PostgreSQL |

Фронтенд: **`REACT_APP_API_URL`** — полный базовый URL API, например `https://<ваш-бэкенд>/api`. При сборке:  
`set REACT_APP_API_URL=...&& npm run build` (Windows) или `REACT_APP_API_URL=... npm run build` (Unix).

## 7. Развёртывание

### 7.1. Локально (Docker)

Команда из корня репозитория: `docker-compose up` (см. `docker-compose.yml`).

### 7.2. Backend (пример: Render)

1. Подключить репозиторий, выбрать корень `backend` (или тот путь, где `manage.py`).  
2. **Build Command:** выполнить скрипт `build.sh` (установка зависимостей, `collectstatic`, `migrate`, при пустой БД — `loaddata fixtures.json`).  
3. **Start Command:** `gunicorn zoo_catalog.wsgi:application --bind 0.0.0.0:$PORT`  
4. Задать `SECRET_KEY`, `DEBUG=False`, `ALLOWED_HOSTS`, `DATABASE_URL`.

### 7.3. Frontend (пример: Netlify)

1. Собрать проект с корректным `REACT_APP_API_URL`.  
2. Публиковать **содержимое каталога `build`**, не весь монорепозиторий.  
3. Файл `frontend/public/_redirects`: правило `/* /index.html 200` для SPA.

### 7.4. Типичные сбои

| Симптом | Причина | Решение |
|---------|---------|---------|
| CORS / Network Error при `POST` | Кастомный заголовок | Добавить `x-session-id` в `CORS_ALLOW_HEADERS` |
| 404 на внутренних маршрутах React | Статический хост | `_redirects` или аналог на хостинге |
| Ошибка модуля WSGI на Render | Неверная точка входа | `zoo_catalog.wsgi:application`, не шаблонное имя |
| Пустой каталог после деплоя фронта | Неверная папка артефакта | Заливать `frontend/build` |

*(Публичные URL в отчёте указывайте свои актуальные; в тексте курсовой они могут отличаться.)*
