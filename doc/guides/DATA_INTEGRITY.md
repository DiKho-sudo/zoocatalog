# Целостность данных при обновлении товаров

## Проблема

При обновлении товаров из разных источников (Django Admin и команды импорта из 1С) важно не затереть данные, которые были изменены вручную администратором.

## Как это работает

### ❌ Неправильно (старый код):

```python
product = Product.objects.get(name=name)
product.price = new_price
product.save()  # Сохраняет ВСЕ поля!
```

**Проблема**: Сохраняются ВСЕ поля товара, даже те, которые не изменялись. Если администратор изменил описание товара, а потом запустился импорт из 1С, описание перезапишется старым значением.

### ✅ Правильно (новый код):

```python
product = Product.objects.get(name=name)
product.price = new_price
product.save(update_fields=['price'])  # Сохраняет ТОЛЬКО цену!
```

**Решение**: Используем параметр `update_fields`, чтобы обновлять только те поля, которые действительно изменились.

---

## Команды импорта

### 1. `sync_stock_prices` - Синхронизация цен и остатков из 1С

**Обновляет только**:
- `price` - цена товара (рассчитывается как стоимость / остаток)
- `stock_status` - статус наличия (в наличии / нет в наличии)

**НЕ затрагивает**:
- Название товара
- Описание
- Категорию
- Производителя
- Изображения
- Все остальные поля

```bash
python manage.py sync_stock_prices stock_price.xlsx
```

---

### 2. `update_prices` - Обновление цен из Excel

**Обновляет только**:
- `price` - цена товара

**НЕ затрагивает**: все остальные поля

```bash
python manage.py update_prices prices.xlsx
```

---

### 3. `import_from_1c` - Полный импорт товаров из 1С

**Создаёт новые товары** или **обновляет существующие полностью**.

⚠️ **Внимание**: Эта команда обновляет ВСЕ поля товара, включая описание, категорию и т.д.

Используйте эту команду только для:
- Первоначального импорта товаров
- Полного обновления каталога

```bash
python manage.py import_from_1c products.xlsx
```

## Сценарии использования

### Сценарий 1: Ежедневное обновление цен и остатков

1. Выгрузите отчёт "Оценка склада" из 1С → `stock_price.xlsx`
2. Запустите:
   ```bash
   python manage.py sync_stock_prices stock_price.xlsx
   ```
3. **Результат**: Обновятся только цены и статусы наличия

✅ **Безопасно**: Не затронет описания, категории и другие данные, которые были изменены вручную в Django Admin.

---

### Сценарий 2: Редактирование товара в Django Admin

1. Администратор заходит в Django Admin
2. Редактирует описание товара, добавляет фото
3. Сохраняет изменения
4. Через час запускается `sync_stock_prices` из 1С
5. **Результат**: Описание и фото остаются, обновляется только цена и статус

✅ **Безопасно**: Изменения в Django Admin сохраняются.

---

### Сценарий 3: Полное обновление каталога

Если в 1С изменились названия товаров, категории, добавились новые товары:

1. Выгрузите полный каталог из 1С → `products.xlsx`
2. Запустите:
   ```bash
   python manage.py import_from_1c products.xlsx
   ```
3. **Результат**: Все товары обновятся полностью

⚠️ **Внимание**: Описания и фото, добавленные вручную в Django Admin, будут перезаписаны данными из 1С.

## Технические детали

### Django ORM `update_fields`

```python
# Сохранить только указанные поля
product.save(update_fields=['price', 'stock_status'])

# SQL запрос:
# UPDATE products_product 
# SET price = 150.00, stock_status = 'in_stock' 
# WHERE id = 123
```

Это генерирует SQL запрос, который обновляет ТОЛЬКО указанные поля, игнорируя все остальные.

### Проверка изменений

Перед применением изменений всегда используйте флаг `--dry-run`:

```bash
python manage.py sync_stock_prices stock_price.xlsx --dry-run
```

Это покажет, что будет изменено, но не сохранит изменения в БД.
