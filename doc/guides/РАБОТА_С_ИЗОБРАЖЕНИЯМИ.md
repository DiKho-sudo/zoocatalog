# 📸 Полное руководство по работе с изображениями

## 🎯 Как работают изображения в проекте

### 1. Где хранятся изображения

```
backend/
  └── media/           ← Папка для всех загруженных файлов
      └── products/    ← Изображения товаров
          ├── image1.jpg
          ├── image2.png
          └── ...
```

### 2. Как они хранятся в базе данных

**В PostgreSQL таблица `products_product`:**
- Поле `image` содержит **путь** к файлу: `products/image1.jpg`
- НЕ само изображение, а только путь!

**Пример записи в БД:**
```
id: 1
name: "Корм Royal Canin"
image: "products/royal-canin.jpg"  ← Путь к файлу
price: 45.90
```

### 3. Как изображения отображаются на сайте

**Frontend формирует полный URL:**
```javascript
const imageUrl = `http://localhost:8000${product.image}`;
// Результат: http://localhost:8000/media/products/royal-canin.jpg
```

## 📝 3 способа добавления изображений

### Способ 1: Через админ-панель (рекомендуется для начала)

1. **Откройте админку:** http://localhost:8000/admin/products/product/
2. **Найдите товар** (используйте поиск)
3. **Нажмите на название товара**
4. **Загрузите изображение:**
   - В поле "Image" нажмите "Выбрать файл"
   - Выберите картинку
   - Нажмите "Сохранить"

✅ Изображение автоматически:
- Загружается в `backend/media/products/`
- Путь сохраняется в БД
- Отображается на сайте

**Для нескольких изображений:**
- Прокрутите вниз до секции "Product images"
- Нажмите "Добавить еще одно изображение товара"
- Загрузите дополнительные фото
- Сохраните

### Способ 2: Массовая загрузка по названиям

**Подготовка:**
1. Создайте папку `images_zoo` в корне проекта
2. Положите туда изображения
3. Назовите файлы ключевыми словами из товара:

```
images_zoo/
├── royal-canin.jpg      → найдет "Royal Canin..."
├── whiskas.png          → найдет "Whiskas..."
├── совок.jpg            → найдет "Совок для корма..."
├── лоток.png            → найдет "Лоток..."
└── ...
```

**Запуск:**
```bash
cd backend
python manage.py add_images ../images_zoo --by-name
```

**Результат:**
```
Сканируем папку: images_zoo
Найдено изображений: 50

Добавлено: "Royal Canin..." ← royal-canin.jpg
Добавлено: "Совок для корма" ← совок.jpg
...

ИТОГИ:
Добавлено изображений: 48
Товары не найдены: 2
```

### Способ 3: Программный импорт

Если у вас есть ссылки на изображения или они на сервере:

```python
from products.models import Product
from django.core.files import File
from urllib.request import urlretrieve

product = Product.objects.get(id=1)

# Скачать из интернета
urlretrieve('https://example.com/image.jpg', 'temp.jpg')
with open('temp.jpg', 'rb') as f:
    product.image.save('product1.jpg', File(f))
```

## 🖼️ Несколько изображений для одного товара

### Через админку:

1. Откройте товар
2. Прокрутите вниз до **"Product images"**
3. Нажмите "Добавить еще одно изображение товара"
4. Загрузите фото
5. Установите порядок (order: 1, 2, 3...)
6. Сохраните

**Результат:**
- Главное изображение в поле `image`
- Дополнительные в таблице `products_productimage`
- На странице товара появится галерея

### Программно:

```python
from products.models import Product, ProductImage
from django.core.files import File

product = Product.objects.get(id=1)

# Добавляем дополнительные изображения
with open('photo2.jpg', 'rb') as f:
    ProductImage.objects.create(
        product=product,
        image=File(f),
        order=1
    )
```

## 📋 Рекомендации по изображениям

### Технические требования:
- **Формат:** JPG, PNG, WEBP
- **Размер:** 800x800px минимум (лучше 1200x1200px)
- **Вес:** до 500 KB
- **Соотношение:** квадратное (1:1) или близкое

### Качество:
- ✅ Белый или светлый фон
- ✅ Товар по центру
- ✅ Хорошее освещение
- ✅ Четкие детали
- ❌ Без водяных знаков

### Сжатие:
Используйте https://tinypng.com/ для сжатия

## 💾 URL изображений в БД

**PostgreSQL хранит:**
```sql
SELECT id, name, image FROM products_product LIMIT 3;

id | name                    | image
1  | Royal Canin Adult       | products/royal-canin_abc123.jpg
2  | Whiskas влажный корм    | products/whiskas_def456.jpg
3  | Лоток для кошек         | products/lotok_ghi789.jpg
```

**Django автоматически:**
- Генерирует уникальное имя файла
- Сохраняет в `media/products/`
- Записывает путь в БД

**Frontend получает:**
```json
{
  "id": 1,
  "name": "Royal Canin Adult",
  "image": "/media/products/royal-canin_abc123.jpg",
  "price": 45.90
}
```

## 🔄 Обновление изображений

### Заменить изображение:
```bash
python manage.py add_images ../images_zoo --by-name --overwrite
```

Флаг `--overwrite` заменит существующие изображения.

## ❓ FAQ

**Q: Обязательно ли добавлять изображения?**  
A: Нет, товары будут работать без картинок (placeholder).

**Q: Сколько изображений можно добавить к товару?**  
A: 1 главное + неограниченное количество дополнительных.

**Q: Где физически хранятся файлы?**  
A: В `backend/media/products/` на сервере.

**Q: Что если удалить товар?**  
A: Django автоматически удалит связанный файл изображения.
