# 🖼️ Руководство по добавлению изображений

## Способы добавления изображений к товарам

### Способ 1: Через админ-панель Django (по одному товару)

1. Откройте http://localhost:8000/admin/
2. Перейдите в **Products** → **Товары**
3. Найдите нужный товар
4. Нажмите на него для редактирования
5. В поле **"Image"** нажмите **"Выбрать файл"**
6. Выберите изображение
7. Нажмите **"Сохранить"**

**Плюсы:** Просто, наглядно  
**Минусы:** Долго для большого количества товаров

---

### Способ 2: Массовая загрузка по названиям файлов (рекомендуется!) ⭐

Это самый удобный способ для загрузки множества картинок сразу.

#### Шаг 1: Подготовьте изображения

Создайте папку, например `images_products` и положите туда картинки.

**Вариант А: Имена = Названия товаров**
```
images_products/
├── Royal Canin.jpg
├── Whiskas.png
├── Совок для корма.jpg
└── ...
```

**Вариант Б: Имена = Артикулы** (если есть артикулы)
```
images_products/
├── RC001.jpg
├── WH002.png
├── DOG123.jpg
└── ...
```

#### Шаг 2: Запустите команду

**Поиск по названию товара:**
```bash
cd backend
python manage.py add_images путь/к/images_products --by-name
```

**Поиск по артикулу:**
```bash
python manage.py add_images путь/к/images_products
```

**С заменой существующих изображений:**
```bash
python manage.py add_images images_products --by-name --overwrite
```

#### Что произойдет:

```

⏭️  Пропущен "Туалет для животных" (изображение уже есть)
❌ Товар не найден для: старое_фото.jpg

==================================================
ИТОГИ:

⏭️  Пропущено (уже есть): 3

```

## 📝 Советы по именованию файлов

### Для поиска по названию (`--by-name`):

**Хорошо:**
- `Royal Canin.jpg` → найдет товар "ECOLINE ROMEO GREEN..."
- `совок.jpg` → найдет "Совок для корма..."
- `лоток.png` → найдет любой лоток

**Плохо:**
- `IMG_001.jpg` → не найдет (непонятное название)
- `photo.jpg` → не найдет

**Совет:** Используйте ключевые слова из названия товара!

### Для поиска по артикулу:

**Хорошо:**
- `RC001.jpg`
- `WH002.png`
- `TRIXIE-123.jpg`

**Плохо:**
- Пробелы в имени файла (могут быть проблемы)

## 🎨 Рекомендации по изображениям

### Технические требования:
- **Формат:** JPG, PNG, WEBP (рекомендуется JPG)
- **Размер:** 800x800px или больше (оптимально 1000x1000px)
- **Вес файла:** до 500 KB (сжимайте большие изображения)
- **Соотношение:** квадратное (1:1) или близкое к нему

### Качество:
- ✅ Светлый фон (белый идеально)
- ✅ Товар в центре
- ✅ Хорошее освещение
- ✅ Четкое изображение
- ❌ Избегайте водяных знаков
- ❌ Избегайте слишком темных фото

### Сжатие изображений:

Используйте онлайн сервисы:
- https://tinypng.com/ - отличное сжатие
- https://squoosh.app/ - продвинутое сжатие
- https://compressor.io/ - быстрое сжатие

## 🔄 Обновление изображений

Если нужно заменить существующие изображения:

```bash
python manage.py add_images images_products --by-name --overwrite
```

Флаг `--overwrite` заменит все текущие картинки на новые.

## ❓ Часто задаваемые вопросы

**Q: Где хранятся загруженные изображения?**  
A: В папке `backend/media/products/`

**Q: Можно ли загрузить несколько картинок для одного товара?**  
A: Да! В админке есть раздел "Product images" (дополнительные изображения).

**Q: Что если имя файла не совпадает точно?**  
A: С флагом `--by-name` команда ищет частичное совпадение (contains).

**Q: Можно ли использовать свои названия файлов?**  
A: Лучше использовать понятные названия связанные с товаром.
